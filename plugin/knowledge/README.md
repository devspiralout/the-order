# Knowledge

This directory contains learnings accumulated by the agent team across sessions
via the dream consolidation process (`/dream`).

These files are **agent-maintained** — they grow and evolve as the team works.
They complement your project standards (the manual) with field experience.

## Files

Knowledge files are created dynamically based on your team configuration:

| File | Contents |
|------|----------|
| `{agent-name}-patterns.md` | One per technical agent — learnings specific to their domain |
| `standards-map.md` | Which project standards apply where, plus gaps and conflicts |
| `review-lessons.md` | Common review feedback and quality patterns |
| `gotchas.md` | Codebase-specific landmines and non-obvious behaviours |
| `doc-gaps.md` | Project docs that are missing, outdated, or conflicting |

## How it works

1. Agents work on tasks (features, reviews, bugs)
2. At session end, `/dream` reviews what happened
3. Non-obvious, reusable learnings are distilled into these files
4. Future sessions load these files for richer context

Knowledge here should be things that **can't be derived** from reading your
project docs alone — the kind of thing a teammate would tell you over coffee.
